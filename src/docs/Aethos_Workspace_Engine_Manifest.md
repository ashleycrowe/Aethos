# Aethos Workspace Engine: Nexus v2.0 Manifest

## 1. Feature Summary: The Intelligence Layer
The Workspace Engine represents a fundamental shift from "File Management" to **"Operational Architecture."** It synthesizes metadata from multiple providers into unified, context-rich environments.

| Feature Area | Component | Operational Purpose |
| :--- | :--- | :--- |
| **Nexus Wizard** | 4-Step Crystallization | Enforces the "Hybrid Anchor" strategy, ensuring every workspace has a Primary Storage Root (M365/Slack) for sovereign write-back while allowing Tier 2 (Shadow) artifacts to be synthesized. |
| **Flight Deck** | Intelligence Dashboard | Shifts focus from file lists to "Pinned Artifacts." Prioritizes high-signal items with **Aethos Notes**, providing narrative context that raw file explorers lack. |
| **Voyager Map** | Constellation UI | A cinematic visual engine that represents metadata nodes as stars. Allows "Operational Architects" to see the density and risk of storage clusters across the enterprise. |
| **Universal Pinning** | Context Extraction | Allows users to "extract signal from noise" by pinning specific files, folders, or Slack channels from any provider into a single unified Nexus view. |
| **Adaptive Cockpit** | UI Personalization | Supports **Bunker Mode** (Deep Space) for focus and **Daylight Mode** for field clarity, including client branding and adaptive accent tinting for multi-tenant environments. |
| **The Oracle** | Federated Query | An AI-powered intelligence layer that queries metadata across all connected adapters (M365, Slack, Google, Box) to provide "Operational Clarity" instead of technical logs. |

## 2. Go-To-Market & Pricing Strategy

### Positioning: The "Operational Architect" Pivot
We are moving away from "Security Janitor" (reactive risk) to "Operational Architect" (proactive clarity). 

*   **The Hook:** Use the "Storage Waste & Governance" dashboard (Shadow Discovery) as the entry point (the "Security Janitor" utility).
*   **The Value:** Upsell the Workspace Engine as the "Operational Cockpit."
*   **The Narrative:** We aren't replacing SharePoint or Slack; we are the **Intelligence Layer** that gives them meaning.

### Pricing Tiers (The "Nexus" Model)
*   **Aethos Discovery (Freemium):** Tier 2 "Shadow Discovery" only. Identifies storage waste and leakage in Google/Box/Local. 
*   **Aethos Architect (Standard):** Full management of M365 & Slack. Access to the Nexus Wizard. Limit of 10 Active Nexuses. 
*   **Aethos Intelligence (Enterprise):** Unlimited Nexuses. Full Oracle AI integration. Custom Client Branding. Priority "Starlight" signals.

## 3. Product Roadmap: Predictive Operations

### Short Term (Next 3 Months)
*   **Remediation Simulations:** Allow Architects to "Dry Run" archival processes across M365 and Box simultaneously.
*   **Nexus Sharing:** Secure collaboration on Nexuses with external partners without granting full tenant access.

### Medium Term (6-9 Months)
*   **Predictive Synthesis:** The Oracle proactively suggests creating a Nexus when it detects high-velocity collaboration between a Slack channel and a SharePoint folder.
*   **Universal Search 2.0:** Vector-based semantic search across all connected "Shadow" and "Core" providers.

### Long Term (12 Months+)
*   **Aethos Learning:** Analyze "Crystallization Patterns" to suggest organizational restructuring based on how work actually happens, not the org chart.
*   **Autonomous Archival:** AI-driven archival based on operational "Half-Life" of project artifacts.
