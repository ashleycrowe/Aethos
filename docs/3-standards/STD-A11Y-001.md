# [STANDARD] Aethos Accessibility Standards (WCAG 2.1 AA)
## Universal Access for the Aethos Intelligence Layer

**Version:** 2.0  
**Date:** February 27, 2026  
**Status:** 🟢 ACTIVE  
**Owner:** UX & Engineering Team  
**Description:** WCAG 2.1 AA compliance standards for the Aethos cinematic glassmorphism interface.  
**Authority:** MANDATORY (Required for Microsoft AppSource)  
**Document ID:** STD-A11Y-001

---

## 🌌 The Aethos Glass Challenge

Aethos uses a **cinematic glassmorphism aesthetic** with deep space backgrounds (#0B0F19). Standard glass techniques often fail WCAG 2.1 AA contrast requirements. This standard mandates specific adaptations to ensure Aethos is accessible to all users while maintaining its visual identity.

**Design Philosophy:**
- Accessibility is non-negotiable
- Beauty and usability are not mutually exclusive
- All users deserve operational clarity

---

## 🚨 MANDATORY CRITICAL RULES

1. ✅ **GLASS CONTRAST (4.5:1)** - Any surface containing body text MUST have an underlying semi-opaque layer (minimum `bg-[#0B0F19]/60`) to ensure readability against dynamic backgrounds.

2. ✅ **CYAN FOCUS GLOW** - Default browser focus rings are forbidden. All interactive elements must use:
   ```css
   ring-2 ring-[#00F0FF] ring-offset-2 ring-offset-[#0B0F19]
   ```

3. ✅ **LIVE REGIONS** - All asynchronous updates (sync status, notifications) MUST use `aria-live="polite"` or `aria-live="assertive"` for critical alerts.

4. ✅ **KEYBOARD ESCAPE** - All modals, drawers, and overlays MUST:
   - Close on `Escape` key
   - Trap focus within the container
   - Return focus to trigger element on close

5. ✅ **SCREEN READER LABELS** - Icon-only buttons MUST have both `aria-label` and visible text (via `<span className="sr-only">`)

6. ✅ **MOTION RESPECT** - All animations must respect `prefers-reduced-motion: reduce`

## ⌨️ PART 1: KEYBOARD NAVIGATION

### 1.1 Global Keyboard Shortcuts

**Implemented Shortcuts:**
| Key | Action | Context |
| :--- | :--- | :--- |
| `/` | Focus Oracle search | Global |
| `Escape` | Close modal/drawer | When modal open |
| `Tab` | Navigate forward | Global |
| `Shift + Tab` | Navigate backward | Global |
| `Enter` | Activate button/link | Focused element |
| `Space` | Toggle checkbox/button | Focused element |
| `Arrow Keys` | Navigate lists | List context |

**Implementation Example:**
```typescript
// Global keyboard shortcuts
useEffect(() => {
  const handleKeyDown = (e: KeyboardEvent) => {
    if (e.key === '/' && !isInputFocused()) {
      e.preventDefault();
      oracleSearchRef.current?.focus();
    }
  };
  
  document.addEventListener('keydown', handleKeyDown);
  return () => document.removeEventListener('keydown', handleKeyDown);
}, []);
```

### 1.2 Focus Management Patterns

**Modal/Drawer Focus Trap:**
```typescript
import { useEffect, useRef } from 'react';

function Modal({ isOpen, onClose, children }: ModalProps) {
  const modalRef = useRef<HTMLDivElement>(null);
  const previousFocusRef = useRef<HTMLElement | null>(null);
  
  useEffect(() => {
    if (isOpen) {
      // Save current focus
      previousFocusRef.current = document.activeElement as HTMLElement;
      
      // Lock body scroll
      document.body.style.overflow = 'hidden';
      
      // Focus first focusable element
      const focusable = modalRef.current?.querySelectorAll(
        'button:not([disabled]), [href], input:not([disabled]), select:not([disabled]), textarea:not([disabled]), [tabindex]:not([tabindex="-1"])'
      );
      (focusable?.[0] as HTMLElement)?.focus();
      
      // Escape key handler
      const handleEscape = (e: KeyboardEvent) => {
        if (e.key === 'Escape') onClose();
      };
      document.addEventListener('keydown', handleEscape);
      
      return () => {
        document.body.style.overflow = '';
        document.removeEventListener('keydown', handleEscape);
        previousFocusRef.current?.focus();
      };
    }
  }, [isOpen, onClose]);
  
  return (
    <div ref={modalRef} role="dialog" aria-modal="true">
      {children}
    </div>
  );
}
```

**Skip Links:**
```tsx
// Must be first element in <body>
<a
  href="#main-content"
  className="sr-only focus:not-sr-only focus:absolute focus:top-4 focus:left-4 focus:z-50 focus:px-4 focus:py-2 focus:bg-[#00F0FF] focus:text-black focus:rounded-lg"
>
  Skip to main content
</a>

// Target element
<main id="main-content" tabIndex={-1}>
  {/* Main content */}
</main>
```

### 1.3 Tab Order Requirements

**Visual Hierarchy = Tab Order:**
1. Skip link (hidden until focused)
2. Sidebar navigation
3. Top header (search, notifications, settings)
4. Main content area
5. Modals/drawers (when open)
6. Toast notifications (when present)

**Testing:**
- Tab through entire interface without mouse
- Ensure no focus traps (except intentional modal traps)
- Verify focus indicator is always visible

## 🔊 PART 2: SCREEN READER (ARIA) COMPLIANCE

### 2.1 Semantic HTML Requirements

**Use semantic HTML first, ARIA second:**
```tsx
// ✅ GOOD: Semantic HTML
<button onClick={handleClick}>Delete</button>
<nav aria-label="Main navigation">...</nav>
<main>...</main>
<aside aria-label="Sidebar">...</aside>

// ❌ BAD: Div soup
<div onClick={handleClick} role="button">Delete</div>
<div role="navigation">...</div>
```

### 2.2 ARIA Labels for Metadata Context

**Entity Pointer Labels (Aethos-Specific):**
Since Aethos displays metadata pointers, labels must provide full context:

```tsx
// ✅ GOOD: Full context
<button
  aria-label="Archive Marketing Legacy SharePoint site, 142 GB, inactive for 2 years"
  onClick={handleArchive}
>
  <Archive className="w-4 h-4" />
  <span className="sr-only">Archive site</span>
</button>

// ❌ BAD: Insufficient context
<button aria-label="Archive" onClick={handleArchive}>
  <Archive className="w-4 h-4" />
</button>
```

**Workspace Intelligence Scores:**
```tsx
<div aria-label={`Workspace intelligence score: ${score} out of 100`}>
  <CircularProgress value={score} />
  <span className="sr-only">{score}/100</span>
</div>
```

### 2.3 Live Regions for Async Updates

**Sync Status (NexusSync component):**
```tsx
<div
  role="status"
  aria-live="polite"
  aria-atomic="true"
  className="flex items-center gap-2"
>
  {status === 'saving' && (
    <>
      <Loader2 className="w-3 h-3 animate-spin" />
      <span>Syncing Nexus metadata...</span>
    </>
  )}
  {status === 'saved' && (
    <>
      <Cloud className="w-3 h-3 text-[#00F0FF]" />
      <span>Nexus pointers secured</span>
    </>
  )}
</div>
```

**Toast Notifications:**
```tsx
import { toast } from 'sonner';

// Accessible toast
toast.success('Workspace created', {
  description: 'Auto-sync enabled with 2 rules',
  duration: 5000,
  // Sonner automatically uses role="status" aria-live="polite"
});

// Critical alert
toast.error('Sync failed', {
  description: 'Unable to connect to Microsoft Graph',
  duration: 10000,
  // Use aria-live="assertive" for errors
});
```

### 2.4 Form Accessibility

**Required Fields:**
```tsx
<label htmlFor="workspace-name" className="text-sm font-bold">
  Workspace Name
  <span aria-label="required" className="text-[#FF5733] ml-1">*</span>
</label>
<input
  id="workspace-name"
  name="name"
  required
  aria-required="true"
  aria-invalid={!!errors.name}
  aria-describedby={errors.name ? "name-error" : undefined}
/>
{errors.name && (
  <span id="name-error" role="alert" className="text-[#FF5733] text-xs">
    {errors.name}
  </span>
)}
```

## 🎨 PART 3: COLOR & CONTRAST (Aethos Glass Spec)

### 3.1 Text Contrast Requirements

**WCAG 2.1 AA Standards:**
- Normal text (< 18pt): 4.5:1 minimum
- Large text (≥ 18pt or bold ≥ 14pt): 3:1 minimum
- Interactive elements: 3:1 minimum

**Aethos Glass Color Tokens:**

| Element | Text Color | Background | Contrast Ratio | WCAG Result |
| :--- | :--- | :--- | :--- | :--- |
| Primary Text | `#FFFFFF` | `bg-[#0B0F19]/60` | 8.2:1 | ✅ Pass AAA |
| Secondary Text | `#94A3B8` (slate-400) | `bg-[#0B0F19]/80` | 5.1:1 | ✅ Pass AA |
| Cyan Action | `#00F0FF` | `bg-[#0B0F19]/40` | 9.5:1 | ✅ Pass AAA |
| Orange Alert | `#FF5733` | `bg-white/5` | 4.8:1 | ✅ Pass AA |
| Links | `#00F0FF` | `bg-transparent` | 11.2:1 | ✅ Pass AAA |

**Testing Tool:**
```bash
# Use axe DevTools browser extension
# Or automated with jest-axe
npm run test:a11y
```

### 3.2 Color Independence

**Never rely on color alone:**
```tsx
// ✅ GOOD: Color + icon + text
<div className="flex items-center gap-2 text-[#FF5733]">
  <AlertCircle className="w-4 h-4" />
  <span>High priority</span>
</div>

// ❌ BAD: Color only
<div className="text-[#FF5733]">
  High priority
</div>
```

### 3.3 Focus Indicators

**Aethos Focus Style (Mandatory):**
```css
/* Global focus styles */
*:focus-visible {
  outline: 2px solid #00F0FF;
  outline-offset: 2px;
  border-radius: 4px;
}

/* Alternative: Tailwind classes */
.focus-aethos {
  @apply focus-visible:ring-2 focus-visible:ring-[#00F0FF] focus-visible:ring-offset-2 focus-visible:ring-offset-[#0B0F19];
}
```

**Example Implementation:**
```tsx
<button className="px-4 py-2 bg-[#00F0FF] text-black rounded-xl focus-visible:ring-2 focus-visible:ring-white focus-visible:ring-offset-2 focus-visible:ring-offset-[#00F0FF]">
  Create Workspace
</button>
```

## 🎬 PART 4: MOTION & ANIMATION

### 4.1 Respecting User Preferences

**Mandatory `prefers-reduced-motion` support:**
```tsx
import { motion } from 'motion/react';

const MotionCard = motion.div;

function AnimatedCard({ children }: { children: React.ReactNode }) {
  const prefersReducedMotion = window.matchMedia(
    '(prefers-reduced-motion: reduce)'
  ).matches;
  
  return (
    <MotionCard
      initial={prefersReducedMotion ? {} : { opacity: 0, y: 20 }}
      animate={prefersReducedMotion ? {} : { opacity: 1, y: 0 }}
      transition={prefersReducedMotion ? { duration: 0 } : { duration: 0.3 }}
    >
      {children}
    </MotionCard>
  );
}
```

**CSS Alternative:**
```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

### 4.2 Safe Animation Practices

**Avoid:**
- Flashing content (> 3 flashes per second)
- Parallax scrolling without disable option
- Auto-playing videos with audio
- Infinite spinning animations

**Safe Patterns:**
- Fade in/out transitions
- Slide animations (< 500ms duration)
- Scale transforms (subtle, 0.95-1.05 range)
- Opacity changes

## 🧪 PART 5: TESTING & VALIDATION

### 5.1 Automated Testing

**jest-axe Integration:**
```typescript
import { render } from '@testing-library/react';
import { axe, toHaveNoViolations } from 'jest-axe';

expect.extend(toHaveNoViolations);

describe('WorkspaceCreationWizard accessibility', () => {
  it('has no accessibility violations', async () => {
    const { container } = render(<WorkspaceCreationWizard />);
    const results = await axe(container);
    expect(results).toHaveNoViolations();
  });
});
```

**Playwright A11y Tests:**
```typescript
import { test, expect } from '@playwright/test';
import AxeBuilder from '@axe-core/playwright';

test('Oracle search page is accessible', async ({ page }) => {
  await page.goto('http://localhost:3000/oracle');
  
  const accessibilityScanResults = await new AxeBuilder({ page })
    .withTags(['wcag2a', 'wcag2aa', 'wcag21a', 'wcag21aa'])
    .analyze();
  
  expect(accessibilityScanResults.violations).toEqual([]);
});
```

### 5.2 Manual Testing Checklist

**Keyboard Navigation:**
- [ ] Can navigate entire app using only Tab/Shift+Tab
- [ ] Focus indicator is always visible
- [ ] Modals trap focus correctly
- [ ] Escape key closes modals/drawers
- [ ] Skip link works and is visible when focused

**Screen Reader Testing:**
- [ ] Test with NVDA (Windows) or VoiceOver (Mac)
- [ ] All interactive elements are announced correctly
- [ ] Headings structure is logical (h1 → h2 → h3)
- [ ] ARIA live regions announce status changes
- [ ] Form errors are announced

**Visual Testing:**
- [ ] Text contrast meets 4.5:1 minimum
- [ ] Color is never the only indicator
- [ ] Focus indicators are visible on all interactive elements
- [ ] Zoom to 200% - no horizontal scrolling, content is readable
- [ ] Test in Windows High Contrast Mode

**Motion Testing:**
- [ ] Enable "Reduce motion" in OS settings
- [ ] Verify animations are disabled or minimal
- [ ] No content loss when animations are disabled
