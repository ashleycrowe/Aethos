## 📋 Executive Summary

This standard governs all interactions between Aethos and Microsoft 365 tenants. **Microsoft 365 is the Tier 1 Primary Anchor of the Aethos Intelligence Layer.** As such, it is the only provider (alongside Slack) authorized for full bi-directional remediation and metadata management. We prioritize metadata efficiency, aggressive caching, and absolute adherence to Microsoft's rate-limiting and security protocols.

**Architecture Context:** Vercel Serverless Functions + MSAL.js. See `/docs/2-ARCHITECTURE/SIMPLIFIED_ARCHITECTURE.md`

---

## 🚨 MANDATORY CRITICAL RULES

1. **GRAPH API ONLY:** Use Microsoft Graph API for all M365 interactions. Avoid deprecated REST endpoints.

2. **GRAPH BATCHING:** Always batch Graph API calls for fetching multiple items (Max 20 requests per batch).

3. **EXPONENTIAL BACKOFF:** Implement circuit breakers and exponential backoff for `429` (Throttled) responses as per `STD-ERROR-001.md`.

4. **LEAST PRIVILEGE:** Permission scopes must be restricted to the absolute minimum required for metadata pointers (delegated permissions only, no application permissions for MVP).

5. **NO CLIENT SECRETS IN FRONTEND:** Client secrets MUST only reside in Vercel Environment Variables and backend functions. Never in frontend code.

6. **AGGRESSIVE CACHING:** Cache Graph metadata:
   - User Profiles: 1 hour
   - Org Structure: 24 hours
   - Photos: 7 days
   - Site Metadata: 4 hours

7. **DELTA QUERIES:** Use delta queries for incremental syncs (sites, files, users) to minimize API calls and respect rate limits.

## 🏗️ Architecture: The Sidecar Pattern

Aethos utilizes a "Sidecar" architecture where the frontend communicates with Vercel Functions, which then interface with M365:

```
┌─────────────────────────────────────────────┐
│ Frontend (React + MSAL.js)                  │
│ - User authentication (delegated)           │
│ - Access token management                   │
│ - Direct Graph calls (read-only metadata)   │
└────────────────┬────────────────────────────┘
                 │ HTTPS
                 ▼
┌─────────────────────────────────────────────┐
│ Vercel Serverless Functions                 │
│ - Heavy lifting (batch operations)          │
│ - Delta sync orchestration                  │
│ - Rate limiting & circuit breakers          │
└────────────────┬────────────────────────────┘
                 │ Graph API
                 ▼
┌─────────────────────────────────────────────┐
│ Microsoft Graph API                          │
│ - Metadata endpoints only                   │
│ - Sites, Files, Users, Groups               │
│ - No file content download                  │
└─────────────────────────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────────┐
│ Supabase PostgreSQL                          │
│ - Store metadata pointers                   │
│ - Cache Graph responses (with TTL)          │
│ - Delta links for incremental sync          │
└─────────────────────────────────────────────┘
```

**Key Points:**
- Frontend: Requests metadata via Aethos UI, uses MSAL.js for auth
- Backend: Vercel Functions orchestrate Graph batches, manage rate limits
- Database: Supabase stores metadata pointers and intelligence scores (never file bodies)

## 📡 API Implementation Standards

### Microsoft Graph API (v1.0)

**Primary Endpoint:** `https://graph.microsoft.com/v1.0/`

**Core Resources:**
- **Sites:** `/sites` - SharePoint site metadata
- **Drives:** `/sites/{siteId}/drives` - Document libraries
- **Lists:** `/sites/{siteId}/lists` - SharePoint lists
- **Users:** `/users` - User profiles (for org structure)
- **Groups:** `/groups` - Teams and M365 Groups

**Usage Guidelines:**

```typescript
// /src/app/services/microsoft/graphClient.ts

import { Client } from '@microsoft/microsoft-graph-client';
import { resilientFetch } from '../api/resilientClient';

export function createGraphClient(accessToken: string) {
  return Client.init({
    authProvider: (done) => {
      done(null, accessToken);
    }
  });
}

// Fetch sites with proper error handling
export async function fetchSites(accessToken: string) {
  const client = createGraphClient(accessToken);
  
  try {
    const response = await client
      .api('/sites')
      .version('v1.0')
      .select('id,displayName,webUrl,createdDateTime,lastModifiedDateTime')
      .top(999)
      .get();
    
    return response.value;
  } catch (error) {
    if (error.statusCode === 429) {
      // Handle throttling (see STD-ERROR-001.md)
      const retryAfter = parseInt(error.headers['Retry-After'] || '60');
      await sleep(retryAfter * 1000);
      return fetchSites(accessToken); // Retry
    }
    throw error;
  }
}
```

### Graph API Batching

**Batch multiple requests to reduce round trips:**

```typescript
// /src/app/services/microsoft/graphBatch.ts

interface BatchRequest {
  id: string;
  method: string;
  url: string;
}

export async function batchGraphRequests(
  accessToken: string,
  requests: BatchRequest[]
): Promise<any[]> {
  // Graph API limits batches to 20 requests
  const batches = chunk(requests, 20);
  const results: any[] = [];
  
  for (const batch of batches) {
    const response = await fetch('https://graph.microsoft.com/v1.0/$batch', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${accessToken}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        requests: batch
      })
    });
    
    const data = await response.json();
    results.push(...data.responses);
  }
  
  return results;
}

// Usage: Fetch multiple users in one call
const userRequests = userIds.map((id, index) => ({
  id: String(index),
  method: 'GET',
  url: `/users/${id}`
}));

const userProfiles = await batchGraphRequests(accessToken, userRequests);
```

### Delta Queries (Incremental Sync)

**Use delta queries to sync only changed items:**

```typescript
// /src/app/services/microsoft/deltaSync.ts

export async function syncSitesDelta(
  accessToken: string,
  tenantId: string
): Promise<void> {
  // Get last delta link from database
  const { data: lastSync } = await supabase
    .from('sync_jobs')
    .select('delta_link')
    .eq('tenant_id', tenantId)
    .eq('job_type', 'sites_delta')
    .order('created_at', { ascending: false })
    .limit(1)
    .single();
  
  const url = lastSync?.delta_link || 'https://graph.microsoft.com/v1.0/sites/delta';
  
  let nextLink: string | null = url;
  let deltaLink: string | null = null;
  
  while (nextLink) {
    const response = await fetch(nextLink, {
      headers: {
        'Authorization': `Bearer ${accessToken}`
      }
    });
    
    const data = await response.json();
    
    // Process changed sites
    const sites = data.value.map(site => ({
      tenant_id: tenantId,
      provider: 'microsoft',
      provider_id: site.id,
      name: site.displayName,
      url: site.webUrl,
      last_modified_at: site.lastModifiedDateTime,
      synced_at: new Date()
    }));
    
    // Upsert to database
    await supabase
      .from('containers')
      .upsert(sites, {
        onConflict: 'tenant_id,provider,provider_id'
      });
    
    // Check for next page or delta link
    nextLink = data['@odata.nextLink'] || null;
    deltaLink = data['@odata.deltaLink'] || null;
  }
  
  // Store delta link for next sync
  if (deltaLink) {
    await supabase
      .from('sync_jobs')
      .insert({
        tenant_id: tenantId,
        job_type: 'sites_delta',
        status: 'completed',
        delta_link: deltaLink,
        completed_at: new Date()
      });
  }
}
```

### Token Management

```typescript
// /src/app/services/microsoft/tokenManager.ts

export async function getAccessToken(): Promise<string> {
  const accounts = msalInstance.getAllAccounts();
  
  if (accounts.length === 0) {
    throw new Error('No user logged in');
  }
  
  try {
    // Try silent token acquisition first
    const response = await msalInstance.acquireTokenSilent({
      ...loginRequest,
      account: accounts[0]
    });
    
    return response.accessToken;
  } catch (error) {
    // Silent acquisition failed, redirect to login
    await msalInstance.acquireTokenRedirect(loginRequest);
    throw new Error('Redirecting to login');
  }
}

// Usage in components
import { getAccessToken } from '@/app/services/microsoft/tokenManager';

async function fetchMyData() {
  const token = await getAccessToken();
  const data = await fetchSites(token);
  return data;
}
```

### Token Storage

**Security Rules:**
- ✅ Use `sessionStorage` for MSAL cache (cleared on browser close)
- ❌ Never use `localStorage` (persists indefinitely, higher risk)
- ✅ Use HTTP-only cookies for refresh tokens (backend only)
- ❌ Never expose access tokens in URL parameters

## 📊 Rate Limiting & Throttling

### Microsoft Graph Limits

| Resource | Limit | Timeframe |
|----------|-------|-----------|
| **Overall** | 10,000 requests | 10 minutes per app |
| **Sites** | 2,000 requests | 10 minutes |
| **Files** | 2,000 requests | 10 minutes |
| **Users** | 2,000 requests | 10 minutes |

**Throttling Responses:**
- **429 Too Many Requests:** Wait `Retry-After` seconds before retrying
- **503 Service Unavailable:** Temporary service issue, retry with backoff
- **504 Gateway Timeout:** Request timeout, retry with backoff

### Handling Throttling

```typescript
// /src/app/services/microsoft/graphHelpers.ts

export async function graphRequestWithRetry<T>(
  fn: () => Promise<T>,
  retries: number = 3
): Promise<T> {
  for (let attempt = 0; attempt <= retries; attempt++) {
    try {
      return await fn();
    } catch (error: any) {
      if (error.statusCode === 429) {
        const retryAfter = parseInt(error.headers?.['Retry-After'] || '60');
        console.log(`Throttled. Waiting ${retryAfter}s before retry...`);
        await sleep(retryAfter * 1000);
        continue;
      }
      
      if (attempt < retries && [503, 504].includes(error.statusCode)) {
        const delay = Math.pow(2, attempt) * 1000;
        console.log(`Server error. Retrying in ${delay}ms...`);
        await sleep(delay);
        continue;
      }
      
      throw error;
    }
  }
  
  throw new Error('Max retries exceeded');
}

// Usage
const sites = await graphRequestWithRetry(() => 
  client.api('/sites').get()
);
```

## 💾 Caching Strategy

### Cache Durations by Resource

| Resource | TTL | Reason |
|----------|-----|--------|
| **User Profiles** | 1 hour | Names/emails change infrequently |
| **Org Structure** | 24 hours | Reporting relationships stable |
| **User Photos** | 7 days | Profile photos rarely change |
| **Site Metadata** | 4 hours | Site properties change moderately |
| **File Metadata** | 1 hour | Files change frequently |
| **Presence** | 5 minutes | User status changes often |

### Cache Implementation

```typescript
// /src/app/services/microsoft/graphCache.ts

interface CacheEntry<T> {
  data: T;
  timestamp: number;
  ttl: number;
}

class GraphCache {
  private cache = new Map<string, CacheEntry<any>>();
  
  set<T>(key: string, data: T, ttl: number): void {
    this.cache.set(key, {
      data,
      timestamp: Date.now(),
      ttl
    });
  }
  
  get<T>(key: string): T | null {
    const entry = this.cache.get(key);
    if (!entry) return null;
    
    const age = Date.now() - entry.timestamp;
    if (age > entry.ttl) {
      this.cache.delete(key);
      return null;
    }
    
    return entry.data;
  }
}

export const graphCache = new GraphCache();

// Usage
export async function getCachedUserProfile(userId: string, accessToken: string) {
  const cacheKey = `user:${userId}`;
  const cached = graphCache.get<UserProfile>(cacheKey);
  
  if (cached) {
    return cached;
  }
  
  const client = createGraphClient(accessToken);
  const profile = await client.api(`/users/${userId}`).get();
  
  graphCache.set(cacheKey, profile, 60 * 60 * 1000); // 1 hour
  
  return profile;
}
```

## ✅ Compliance Checklist

Before deploying M365 integration:

- [ ] **Authentication:** MSAL.js configured with correct client ID and tenant ID
- [ ] **Scopes:** Reviewed and minimized (e.g., `User.ReadBasic.All` vs `User.Read.All`)
- [ ] **Batching:** Graph batching implemented for bulk operations (max 20/batch)
- [ ] **Throttling:** Retry-After headers honored for all M365 requests
- [ ] **Delta Queries:** Delta tokens persisted in Supabase for efficient sync
- [ ] **Caching:** Metadata cached with appropriate TTLs
- [ ] **Error Handling:** Circuit breakers and exponential backoff implemented
- [ ] **Security:** No client secrets in frontend code (Vercel env vars only)
- [ ] **Token Storage:** Using sessionStorage, not localStorage
- [ ] **Rate Limiting:** Client-side rate limiter respects Graph API limits
