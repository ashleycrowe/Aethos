## 📋 Executive Summary
Aethos is designed to serve global enterprises. This standard ensures that "The Flashlight" and "The Nexus" are architecturally prepared for international expansion. We prioritize locale-aware intelligence, ensuring that storage waste metrics and virtual workspace metadata are presented in the user's native cultural and linguistic context without degrading the cinematic "Aethos Glass" experience.

---

## 🚨 MANDATORY CRITICAL RULES

1.  **STRING EXTERNALIZATION:** Zero hardcoded user-facing strings in the UI. All text MUST be managed via the i18n translation framework.
2.  **LOCALE-AWARE FORMATTING:** Dates, times, and numbers (especially storage waste bytes) MUST utilize the browser's `Intl` API or locale-aware libraries.
3.  **RTL READINESS:** The "Aethos Glass" design system MUST utilize logical CSS properties (e.g., `margin-inline-start`) to support Right-to-Left (RTL) languages like Arabic and Hebrew.
4.  **NO PAGE RELOADS:** Locale switching must be handled dynamically via React state, maintaining the cinematic continuity of the Sidecar interface.
5.  **HUMAN VALIDATION:** Machine translations are strictly for development staging. All production-ready translations MUST be reviewed by native speakers for cultural nuance.

---

## 🌍 i18n Framework Strategy

### Technology Stack
- **Framework:** `react-i18next`.
- **Backend:** `i18next-http-backend` (Sidecar-hosted translation assets).
- **Detection:** `i18next-browser-languagedetector` (Prioritizing M365 tenant locale).

### Logical Properties (RTL Support)
To maintain visual integrity in RTL markets, use Tailwind v4 logical classes:
- ✅ `ms-4` (Margin Start) instead of `ml-4`.
- ✅ `text-start` instead of `text-left`.
- ✅ `rounded-s-xl` instead of `rounded-l-xl`.

---

## 📅 Locale Intelligence

### Waste Indicators (Supernova Orange)
Storage waste metrics must be formatted according to regional standards:
- **US:** `1.2 TB`
- **EU:** `1,2 TB`

### Time-to-Nexus
Synchronicity indicators in "The Nexus" must reflect the user's local timezone, calculated against the M365 tenant's base UTC timestamp.

## 🔄 Translation Workflow
1.  **Key Extraction:** Automate key extraction from `.tsx` files into `/public/locales/en/`.
2.  **Sidecar Storage:** Store primary translation JSONs in the Azure Static Web App's assets directory.
3.  **Client Review:** Use the "Demo Lab" to preview translations in context before production deployment.

## ✅ Compliance Checklist
- [ ] No hardcoded strings in `/src/app/components`.
- [ ] `Intl.NumberFormat` used for all capacity metrics.
- [ ] `dir="rtl"` logic implemented in the root `App` container.
- [ ] Fallback language (English) configured for all keys.
